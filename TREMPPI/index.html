TREMPPI 0.4.0
Toolkit for Reverse Engineering of Molecular Pathways through Parameter Identification
Introduction

This the homepage of TREMPPI, a systems biology toolkit for research in gene regulation and molecular signalling.

This fully integrated, visual toolkit is well suited for the analysis of complex functional modules, e.g. circadian clock. TREMPPI supports multiple ways of encoding various biological knowledge about the system. The goal is to provide as much information as possible about how the structure of the system is likely to look. Such an approach is most valuable if the precise mechanics of the module in question are not known. In this scenario we simply consider all the possibilities for how the components of the module may interact and analyse them one by one. This is a computationally very demanding task. To this end we are employing a coarse-grained modelling framework of logical networks, focusing solely on qualitative features of a model. For an example of use see [2].

Download & Execution

TREMPPI is currently in Alpha version and the user experience is likely to be rough. The developers bear no liability for any problems connected to using the software.

The easiest way of getting TREMPPI is via pre-built packages:

  • Ubuntu/Debian 32bit (preferred for Ubuntu)
  • Ubuntu/Debian 64bit (preferred for Debian)
  • Windows 32bit
    • You also need the Visual Studio 2015 runtime, if you do not have it already.

For all systems you need the Python 3. On Windows (also on Linux if it is in a non-standard location, the environment points to Python 2, etc.) PYTHONHOME must point to Python 3 location. This can be achieved temporarily from a Unix terminal by calling:

export PYTHONHOME=path_to_python3
path_to_TREMPPI/tremppi

On Windows from the cmd tool by calling:

set PYTHONHOME=path_to_python3
path_to_TREMPPI/tremppi

If your system is not in the list, or the binary does not work (in that case also please consider sharing your experience at estherfi.muni.cz), you can try building it from the source at GITHUB.

Tutorial

TREMPPI has a fully integrated graphical user interface that displays in a browser. It can be used in the same manner as any other webpage. Once the GUI is started, a project is opened and the list of widgets is available. Each of the widgets provides access to some part of the functionality of TREMPPI. These are illustrated in detail in the following section, before that we illustrate the general workflow of TREMPPI step by step.

In this section we present a toy example to demonstrate the key concepts of TREMPPI:

  1. START TREMPPI: The tool can be started either by double-clicking on the executable named tremppi or from the command line by executing the command from the directory in which it resides. Note that if there is an error stating that a python module is missing, the PYTHONHOME needs to be set as explained above.
  2. OPEN A PROJECT: TREMPPI starts with a single project opened on start, called project_0, residing in the list of projects on the left. Underneath you can find a list of widgets available to each project. The entry point of each project is the "index" widget. Open it by left-click. After opening, you will see a panel with description of the project, which is empty as of now. Write arbitrary text to this windows for now, e.g. "Hello World". After writing the text, click the SAVE button in the bottom-left corner. This will result in your text being stored within the project data files. On the bottom black-line you should see the text "index has saved successfully" as a confirmation. You can also observe that your text has been saved by pressing the reload button (usually F5). After the page reloads, your text should appear again.
  3. BUILD A MODEL: Open the editor widget. Click the Add button on the top and then the grey area underneath. A new component will appear. Repeat. Once you have two components, click add again, then one component, then the other. A regulation between the two appears. Lastly click a component. Its name and maximal value appears on the top panel. Write a name instead of the generic New and confirm with enter. The name in the graph changes as well. Repeat also for the second component and lastly click the SAVE button.
  4. ENUMERATE PARAMETERS: Open the tools widged and click the spawn button.
  5. GET BASIC STATISTICS: Click the qualitative and quantitative buttons.
  6. READ THE STATISTICS: Click the qualitative or quantitative widget on the left. A new category, called files, appears with a single element, called all. Left click it to gain basic knowledge about the parametrization space, concluding your introduction to TREMPPI.
Publishing

Once it is finished, a project can be easily published on-line or shared without any further need for TREMPPI itself. To publish the project on a web, simply copy its folder to the target server, it will be immediately available. To share the project, copy the folder too, then it can be viewed in two ways:

  • Open the file "index.html" in a browser. This way is not supported by the Google Chrome browser, due to security restrictions. Possibly some other browsers also may cause problem.
  • Start a local HTML server in the folder, e.g. if Python is available by the command "python SimpleHTTPServer".
Reference

In this section we detail the individual widgets.

Index

This is the entry point of a project. The purpose of this widget is documentation and project manipulation. If TREMPPI is started in folder of a particular project, only the SAVE button is available. If TREMPPI is started in a folder containing folders of other projects, then the following project controls are available:

  • NAME FIELD: Enter a new name into this field to rename the project.
  • CLONE: Creates a new project with the exact contents of this one.
  • RENAME: Changes the name of the project.
  • DELETE: Removes the project and all its data.
Editor

The editor is used to create the regulatory network, see [3] for details.

  • To create a component, click the NEW button and then empty space.
  • To create a regulation, click the NEW button, then the source component and lastly the target component.
  • The component has a Name and a Maximal Activity Level. When you select a component, these appear on the top of the page, where the value can be replaced.
  • Similarly, after selecting an edge, the Threshold value and the edge Label can be selected. The meaning of the labels is explained in detail in [4]
    • .
Note that the changes are applied to graph only after the enter button is pressed.
Select

Selection is one of the core notions of TREMPPI. In the intuitive terms, selection is a condition each model must satisfy to be selected. This is used to label/analyse only certain models in the pool, which is crucial mainly for the analysis step. For each label that is available in the database, there is an option to create a selection condition. First, click add to create a new selection and write a new name for it. Then for each field that is of an interest add the selection conditions. There are three ways of doing so:

  • Explicit value. By writing down an explicit number, you state that all models in the selection must have this label equal to this value. Use the decimal point for the values that are expected to be floating point values.
  • A range, e.g. (0.0,1.0], which states that the value must be between 0 (exclusive) and 1 (inclusive).
  • A member of a list, for those values which provide the capability (e.g. sign).

All the selections that are checked on the (on the left) are applied in disjunction, i.e. a model that meets any picked selections is going to be selected.

Properties

Properties page is used to describe discrete formulas used by the validate tool. The widget is divided into two tables. The bottom table is used to describe the measurements. For each component, you can enter its exact value or a range of values in the given measurement. The delta value states what sort of change can the component undergo, i.e. if it can go only up, down, or can not change at all in the steps from when the current measurement is entered until the next is entered. Note that each measurement has a letter assigned to it.

The upper table holds a list of properties. Each property has following components:

  1. Name: this name is then referred to for the cost and robustness values.
  2. Ending: represents how the model should behave after the last measurement. This can be either left open (any value), or it may be required that it reaches a stable state, or it may be required that from a certain point a measurement gets repeated.
  3. Bound: currently unused.
  4. Name of the component: use a range or a specific value to fix this component to that value / range.
Quantitative

First of the reporting tools. The select and compare buttons pick the data to load. All, Left, Mid, Right select which of the reports are going to be visible at a time. Each of the quantitative labels that is available at the time of creation of the report is depicted. The count value states how many entries attain a non-zero value.

Qualitative

Similar to the quantitative report, however it displays rather how many different qualitative values there are and how big a portion of the selection they represent.

Regulations

An inferred regulatory graph, describing the mean impact in the selection, together with the frequency value (how often does an edge has a non-zero sign) and the sign itself.

The relative button changes the scale for the values to the range of the values in the selection.

Correlations

Similar to the regulations graph, only with the correlation instead of impact.

Tools

This widget provides three panels.

The first provides the list of tools, you can start a tool by clicking on it.

The second shows currently running processes. You can stop them by clicking "kill all".

The third show the log from the processes.
